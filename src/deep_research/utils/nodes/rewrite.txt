# 转写判断原则

**当你认为当前网页内容已经足够规范、精炼，且直接使用其原始文本不会影响后续分析效率时，你可以选择不转写。** 此时，系统将直接处理原始文章内容，以节省Token用量。例如：
- 文章内容高度规范可靠，逻辑结构清晰，例如：权威学术论文（PDF或排版良好的HTML版）、官方技术文档或标准规范和结构化数据表格列表
- 网页纯文本布局整洁，基本没有无关的广告、导航、评论等干扰信息。
- 内容与研究主题高度相关，几乎没有冗余信息。

**当你认为当前网页内容存在冗余、混乱或需要优化以提高信息利用效率时，你应该选择转写。** 转写旨在提炼核心信息，去除干扰，并以更适合后续分析的格式呈现。例如：
- 文章内容较为繁复，含有大量与当前研究问题无关的背景介绍、案例分析、商业推广或个人观点。
- 读取到的网页纯文本较为杂乱，包含大量HTML标签、脚本代码、广告文本、导航链接等非内容信息，影响核心信息的提取。
- 信息分散在网页的不同区域，需要整合归纳。
- 原始文本的结构不适合直接用于后续的知识图谱构建、摘要生成或论证支持。
- 其它你认为通过转写可以显著提高文本质量、精简信息或优化后续处理效率的情况。

**注意，如果文本过长，不再转写（大于十万字）**

最后，无论如何，你都应该**添加一个对文本的简短摘要**，控制字数在200字以内

# 转写规范

如果决定转写，请严格遵循以下规范：

1. **聚焦核心信息：** 仅保留与当前研究主题直接相关、具有实质性价值的信息。
2. **去除冗余：** 删除所有无关的背景信息、广告、导航、评论、重复表述、口水话等，使用简洁、准确、客观的语言重新组织内容。
3. **保持逻辑完整性：** 确保转写后的内容逻辑连贯，信息完整，不遗漏关键论点或数据。
4. **保留关键代码、数据、引用：** 对于重要的代码、数字、事实、定义、引用，务必准确无误地保留。
5. **中立客观：** 绝对禁止在转写过程中引入个人偏见，忠实于原文的核心含义。

# 输出要求

请严格按照以下JSON格式输出决策，不要附加任何额外内容或解释性文字。

**如果决定不转写，只需输出：**

{
    "rewrite": false,
    "summary": "这段文本的摘要，控制字数在200字"
}

**如果决定转写，输出：**

{
    "rewrite": true,
    "content": "your rewritting here",
    "summary": "这段文本的摘要，控制字数在200字"
}
